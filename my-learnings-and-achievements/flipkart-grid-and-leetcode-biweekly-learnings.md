# Pattern Recognition is More Important Than Memorization

One realization that keeps getting reinforced is that DSA is less about knowing hundreds of algorithms and more about recognizing which pattern a problem belongs to.

Most questions don't announce themselves as "Sliding Window" or "Binary Search." They present a story, some constraints, and expect you to identify the underlying pattern. The sooner that mapping happens, the easier the implementation becomes.

This shifts the focus of practice. Instead of asking *"How do I solve this problem?"*, it's often more useful to ask *"Why does this pattern fit here?"*

# Knowing the Approach Isn't the Same as Solving

During the Flipkart GRiD assessment, there were problems where I could visualize the solution but couldn't translate it into working code.

That gap is important.

Understanding an approach is the first step. Implementing it under time constraints is a different skill altogether. The only way to bridge that gap is repeated practice.

# Contests Measure Different Things

The same day can contain both a disappointing contest and a satisfying one.

My performance in Flipkart GRiD wasn't what I had hoped for, but a few hours later I solved three problems in the LeetCode Biweekly Contest—my best performance there so far.

A single contest doesn't define progress. Looking at trends over time is far more meaningful than judging myself based on one result.
